ToLower()
• Converts text to lowercase based on the current system culture (regional rules).
• Meaning: results can change depending on the user's language settings.
• Example: Turkish “I” behaves differently.

ToLowerInvariant()
• Converts text to lowercase using a fixed, culture-independent rule.
• Always gives the same result everywhere.
• Used for IDs, keys, comparisons, URLs, hashing → anything that must be consistent.

Tiny vuln note: using ToLower() in security checks can cause culture-based bypasses. ToLowerInvariant() avoids that.

Tiny vuln note: using Now in auth tokens or expiry logic can cause time-zone based bugs — always use UtcNow for security-related timestamps.

# 📡 Complete HTTP Status Codes Reference Guide

## Overview
HTTP status codes are three-digit numbers returned by servers to indicate the result of a client's request. They are grouped into five categories based on the first digit.

---

## 📊 Categories

| Range | Category | Meaning |
|-------|----------|---------|
| 1xx | Informational | Request received, continuing process |
| 2xx | Success | Request successfully received, understood, and accepted |
| 3xx | Redirection | Further action needed to complete the request |
| 4xx | Client Error | Request contains bad syntax or cannot be fulfilled |
| 5xx | Server Error | Server failed to fulfill a valid request |

---

# 1️⃣ 1xx - Informational Responses
These indicate that the request was received and is being processed.

## **100 Continue**
- **Meaning:** Server has received request headers, client should proceed to send body
- **Use Case:** Large file uploads, client checks if server will accept before sending data
- **Example:**
```http
POST /upload HTTP/1.1
Expect: 100-continue

Response:
HTTP/1.1 100 Continue
```

## **101 Switching Protocols**
- **Meaning:** Server is switching protocols as requested by client
- **Use Case:** Upgrading from HTTP to WebSocket
- **Example:**
```http
GET /chat HTTP/1.1
Upgrade: websocket

Response:
HTTP/1.1 101 Switching Protocols
Upgrade: websocket
```

## **102 Processing** (WebDAV)
- **Meaning:** Server has received and is processing request, but no response yet
- **Use Case:** Long-running operations
- **Note:** Prevents client timeout

## **103 Early Hints**
- **Meaning:** Returns some response headers before final response
- **Use Case:** Preload resources while server prepares response
- **Example:**
```http
HTTP/1.1 103 Early Hints
Link: </style.css>; rel=preload; as=style
```

---

# 2️⃣ 2xx - Success
The request was successfully received, understood, and accepted.

## **200 OK**
- **Meaning:** Request succeeded
- **Use Case:** Most common success response
- **Example:**
```http
GET /users/123 HTTP/1.1

Response:
HTTP/1.1 200 OK
Content-Type: application/json

{"id": 123, "name": "John"}
```

## **201 Created**
- **Meaning:** Request succeeded and new resource was created
- **Use Case:** POST requests that create resources
- **Example:**
```http
POST /users HTTP/1.1

Response:
HTTP/1.1 201 Created
Location: /users/124
```

## **202 Accepted**
- **Meaning:** Request accepted but not yet processed
- **Use Case:** Asynchronous processing, batch operations
- **Example:**
```http
POST /reports/generate HTTP/1.1

Response:
HTTP/1.1 202 Accepted
Location: /reports/status/456
```

## **203 Non-Authoritative Information**
- **Meaning:** Request successful but content may be from third-party
- **Use Case:** Proxy servers modifying response
- **Rarely Used**

## **204 No Content**
- **Meaning:** Request successful but no content to return
- **Use Case:** DELETE operations, PUT updates without response body
- **Example:**
```http
DELETE /users/123 HTTP/1.1

Response:
HTTP/1.1 204 No Content
```

## **205 Reset Content**
- **Meaning:** Request successful, client should reset document view
- **Use Case:** Form submissions that should clear the form
- **Example:** After submitting a survey

## **206 Partial Content**
- **Meaning:** Server is delivering part of resource (range request)
- **Use Case:** Video streaming, large file downloads with resume capability
- **Example:**
```http
GET /video.mp4 HTTP/1.1
Range: bytes=0-1023

Response:
HTTP/1.1 206 Partial Content
Content-Range: bytes 0-1023/5000
```

## **207 Multi-Status** (WebDAV)
- **Meaning:** Multiple status codes for multiple operations
- **Use Case:** Batch operations in WebDAV
- **Example:** Multiple file operations

## **208 Already Reported** (WebDAV)
- **Meaning:** Members already enumerated in previous response
- **Use Case:** Avoid repeatedly listing same resources

## **226 IM Used**
- **Meaning:** Server fulfilled GET request with instance manipulations
- **Use Case:** Delta encoding responses
- **Rarely Used**

---

# 3️⃣ 3xx - Redirection
Further action is needed to complete the request.

## **300 Multiple Choices**
- **Meaning:** Multiple options available for resource
- **Use Case:** Different format options (JSON, XML, HTML)
- **Rarely Used in modern APIs**

## **301 Moved Permanently**
- **Meaning:** Resource permanently moved to new URL
- **Use Case:** Website migrations, URL restructuring
- **SEO Impact:** Search engines update their indexes
- **Example:**
```http
GET /old-page HTTP/1.1

Response:
HTTP/1.1 301 Moved Permanently
Location: /new-page
```
- **Important:** Browsers cache this redirect

## **302 Found** (Temporary Redirect)
- **Meaning:** Resource temporarily at different URL
- **Use Case:** Temporary maintenance pages, A/B testing
- **Example:**
```http
GET /product HTTP/1.1

Response:
HTTP/1.1 302 Found
Location: /maintenance
```
- **Note:** May change request method to GET

## **303 See Other**
- **Meaning:** Response can be found at different URL using GET
- **Use Case:** After POST submission, redirect to GET page
- **Example:**
```http
POST /login HTTP/1.1

Response:
HTTP/1.1 303 See Other
Location: /dashboard
```
- **PRG Pattern:** Post-Redirect-Get

## **304 Not Modified**
- **Meaning:** Resource not modified since last request
- **Use Case:** Caching, conditional requests
- **Example:**
```http
GET /image.jpg HTTP/1.1
If-None-Match: "abc123"

Response:
HTTP/1.1 304 Not Modified
```
- **Performance:** Saves bandwidth

## **305 Use Proxy** (Deprecated)
- **Meaning:** Must access resource through proxy
- **Security Risk:** Deprecated due to security concerns

## **306 (Unused)**
- **Status:** Reserved but no longer used

## **307 Temporary Redirect**
- **Meaning:** Temporary redirect, keep same HTTP method
- **Use Case:** Temporary redirects where method must be preserved
- **Example:**
```http
POST /api/data HTTP/1.1

Response:
HTTP/1.1 307 Temporary Redirect
Location: /api/data-temp
```
- **Difference from 302:** Preserves HTTP method

## **308 Permanent Redirect**
- **Meaning:** Permanent redirect, keep same HTTP method
- **Use Case:** Permanent redirects where method must be preserved
- **Example:** API versioning with preserved methods
- **Difference from 301:** Preserves HTTP method

---

# 4️⃣ 4xx - Client Errors
The request contains bad syntax or cannot be fulfilled.

## **400 Bad Request**
- **Meaning:** Server cannot process due to client error
- **Use Case:** Malformed JSON, invalid parameters, validation errors
- **Example:**
```http
POST /users HTTP/1.1
Content-Type: application/json

{"name": ""}

Response:
HTTP/1.1 400 Bad Request
{"error": "Name is required"}
```

## **401 Unauthorized**
- **Meaning:** Authentication required or failed
- **Use Case:** Missing or invalid credentials
- **Example:**
```http
GET /protected-resource HTTP/1.1

Response:
HTTP/1.1 401 Unauthorized
WWW-Authenticate: Bearer realm="API"
{"error": "Authentication required"}
```
- **Note:** Despite name, means "unauthenticated"

## **402 Payment Required**
- **Meaning:** Reserved for future use
- **Use Case:** Digital payment systems (experimental)
- **Status:** Not widely implemented

## **403 Forbidden**
- **Meaning:** Server understood but refuses to authorize
- **Use Case:** Insufficient permissions, IP blocking
- **Example:**
```http
DELETE /users/admin HTTP/1.1
Authorization: Bearer user-token

Response:
HTTP/1.1 403 Forbidden
{"error": "Insufficient permissions"}
```
- **Difference from 401:** Authenticated but not authorized

## **404 Not Found**
- **Meaning:** Resource does not exist
- **Use Case:** Invalid URLs, deleted resources
- **Example:**
```http
GET /users/99999 HTTP/1.1

Response:
HTTP/1.1 404 Not Found
{"error": "User not found"}
```
- **Most Common:** Client error

## **405 Method Not Allowed**
- **Meaning:** HTTP method not supported for resource
- **Use Case:** Trying DELETE on read-only endpoint
- **Example:**
```http
DELETE /public-data HTTP/1.1

Response:
HTTP/1.1 405 Method Not Allowed
Allow: GET, HEAD, OPTIONS
```

## **406 Not Acceptable**
- **Meaning:** Cannot produce response matching Accept headers
- **Use Case:** Client requests XML but only JSON available
- **Example:**
```http
GET /data HTTP/1.1
Accept: application/xml

Response:
HTTP/1.1 406 Not Acceptable
{"error": "Only JSON format supported"}
```

## **407 Proxy Authentication Required**
- **Meaning:** Client must authenticate with proxy
- **Use Case:** Corporate proxy servers
- **Similar to:** 401 but for proxies

## **408 Request Timeout**
- **Meaning:** Server timed out waiting for request
- **Use Case:** Slow client connections
- **Example:** Client takes too long to send data

## **409 Conflict**
- **Meaning:** Request conflicts with current state
- **Use Case:** Version conflicts, duplicate entries
- **Example:**
```http
PUT /users/123 HTTP/1.1
If-Match: "v1"

Response:
HTTP/1.1 409 Conflict
{"error": "Resource version mismatch", "current_version": "v2"}
```

## **410 Gone**
- **Meaning:** Resource permanently deleted
- **Use Case:** Intentionally removed content
- **Example:**
```http
GET /old-article HTTP/1.1

Response:
HTTP/1.1 410 Gone
{"error": "This content has been permanently removed"}
```
- **Difference from 404:** Known to be permanently gone

## **411 Length Required**
- **Meaning:** Content-Length header required
- **Use Case:** POST/PUT without Content-Length
- **Example:**
```http
POST /upload HTTP/1.1

Response:
HTTP/1.1 411 Length Required
```

## **412 Precondition Failed**
- **Meaning:** Precondition in headers evaluated to false
- **Use Case:** Conditional requests, optimistic locking
- **Example:**
```http
PUT /document HTTP/1.1
If-Unmodified-Since: Wed, 21 Oct 2015 07:28:00 GMT

Response:
HTTP/1.1 412 Precondition Failed
```

## **413 Payload Too Large**
- **Meaning:** Request entity too large
- **Use Case:** File upload exceeds size limit
- **Example:**
```http
POST /upload HTTP/1.1
Content-Length: 50000000

Response:
HTTP/1.1 413 Payload Too Large
{"error": "Maximum file size is 10MB"}
```

## **414 URI Too Long**
- **Meaning:** URI longer than server willing to interpret
- **Use Case:** Excessive query parameters
- **Solution:** Use POST instead of GET

## **415 Unsupported Media Type**
- **Meaning:** Request payload format not supported
- **Use Case:** Sending XML when only JSON accepted
- **Example:**
```http
POST /api/data HTTP/1.1
Content-Type: application/xml

Response:
HTTP/1.1 415 Unsupported Media Type
{"error": "Only application/json is supported"}
```

## **416 Range Not Satisfiable**
- **Meaning:** Range header cannot be satisfied
- **Use Case:** Requested byte range out of bounds
- **Example:**
```http
GET /file.pdf HTTP/1.1
Range: bytes=1000-2000

Response:
HTTP/1.1 416 Range Not Satisfiable
Content-Range: bytes */500
```

## **417 Expectation Failed**
- **Meaning:** Expect header cannot be met
- **Use Case:** Expect: 100-continue not satisfied
- **Rarely Used**

## **418 I'm a teapot**
- **Meaning:** Server refuses to brew coffee (April Fools' joke)
- **Origin:** RFC 2324 (HTCPCP protocol)
- **Use Case:** Easter egg, should not be used in production
- **Example:** Try brewing coffee with teapot

## **421 Misdirected Request**
- **Meaning:** Request directed to server unable to respond
- **Use Case:** HTTP/2 server not authoritative for URL
- **Modern:** HTTP/2 specific

## **422 Unprocessable Entity** (WebDAV)
- **Meaning:** Request well-formed but semantic errors
- **Use Case:** Validation errors, business logic violations
- **Example:**
```http
POST /users HTTP/1.1
{"email": "invalid-email", "age": -5}

Response:
HTTP/1.1 422 Unprocessable Entity
{
  "errors": {
    "email": "Invalid email format",
    "age": "Age must be positive"
  }
}
```
- **Popular:** REST APIs for validation errors

## **423 Locked** (WebDAV)
- **Meaning:** Resource is locked
- **Use Case:** File being edited by another user
- **WebDAV Specific**

## **424 Failed Dependency** (WebDAV)
- **Meaning:** Request failed due to failure of previous request
- **Use Case:** Batch operations dependency

## **425 Too Early**
- **Meaning:** Server unwilling to risk processing early data
- **Use Case:** TLS early data protection
- **Security:** Prevents replay attacks

## **426 Upgrade Required**
- **Meaning:** Client should switch to different protocol
- **Use Case:** Force HTTPS, upgrade to HTTP/2
- **Example:**
```http
GET /secure-api HTTP/1.1

Response:
HTTP/1.1 426 Upgrade Required
Upgrade: TLS/1.2, HTTP/2.0
```

## **428 Precondition Required**
- **Meaning:** Server requires conditional request
- **Use Case:** Preventing lost updates
- **Example:**
```http
PUT /document HTTP/1.1

Response:
HTTP/1.1 428 Precondition Required
{"error": "If-Match header required"}
```

## **429 Too Many Requests**
- **Meaning:** Rate limit exceeded
- **Use Case:** API rate limiting, DDoS protection
- **Example:**
```http
GET /api/data HTTP/1.1

Response:
HTTP/1.1 429 Too Many Requests
Retry-After: 3600
X-RateLimit-Limit: 100
X-RateLimit-Remaining: 0
X-RateLimit-Reset: 1640000000
{"error": "Rate limit exceeded. Try again in 1 hour"}
```
- **Important:** Include Retry-After header

## **431 Request Header Fields Too Large**
- **Meaning:** Request headers too large
- **Use Case:** Excessive cookies, large authorization tokens
- **Solution:** Reduce header size or use POST body

## **451 Unavailable For Legal Reasons**
- **Meaning:** Resource unavailable due to legal reasons
- **Use Case:** Censorship, DMCA takedowns, geographic restrictions
- **Example:**
```http
GET /blocked-content HTTP/1.1

Response:
HTTP/1.1 451 Unavailable For Legal Reasons
{"error": "Content blocked due to legal request"}
```
- **Reference:** Fahrenheit 451 (book about censorship)

# 5️⃣ 5xx - Server Errors
The server failed to fulfill a valid request.

## **500 Internal Server Error**
- **Meaning:** Generic server error
- **Use Case:** Unhandled exceptions, crashes, bugs
- **Example:**
```http
GET /api/data HTTP/1.1

Response:
HTTP/1.1 500 Internal Server Error
{"error": "An unexpected error occurred"}
```
- **Security:** Never expose stack traces in production

## **501 Not Implemented**
- **Meaning:** Server does not support functionality
- **Use Case:** Unsupported HTTP method
- **Example:**
```http
PATCH /resource HTTP/1.1

Response:
HTTP/1.1 501 Not Implemented
{"error": "PATCH method not implemented"}
```

## **502 Bad Gateway**
- **Meaning:** Invalid response from upstream server
- **Use Case:** Proxy/gateway received invalid response
- **Example:** Nginx → Backend server returns invalid response
- **Common Causes:**
  - Backend server down
  - Network issues
  - Timeout

## **503 Service Unavailable**
- **Meaning:** Server temporarily unable to handle request
- **Use Case:** Maintenance, overload, temporary downtime
- **Example:**
```http
GET /api HTTP/1.1

Response:
HTTP/1.1 503 Service Unavailable
Retry-After: 120
{"error": "Service temporarily unavailable. Please try again later."}
```
- **Include:** Retry-After header

## **504 Gateway Timeout**
- **Meaning:** Upstream server didn't respond in time
- **Use Case:** Backend server too slow, network timeout
- **Example:** API gateway → Backend takes too long
- **Difference from 502:** Timeout vs invalid response

## **505 HTTP Version Not Supported**
- **Meaning:** HTTP version not supported
- **Use Case:** Old clients using HTTP/0.9
- **Rarely Seen:** Most servers support multiple versions

## **506 Variant Also Negotiates**
- **Meaning:** Server configuration error in content negotiation
- **Use Case:** Circular reference in content negotiation
- **Rare:** Server misconfiguration

## **507 Insufficient Storage** (WebDAV)
- **Meaning:** Server cannot store representation
- **Use Case:** Disk full, quota exceeded
- **Example:**
```http
PUT /large-file HTTP/1.1

Response:
HTTP/1.1 507 Insufficient Storage
{"error": "Storage quota exceeded"}
```

## **508 Loop Detected** (WebDAV)
- **Meaning:** Infinite loop detected processing request
- **Use Case:** Recursive directory operations
- **WebDAV Specific**

## **510 Not Extended**
- **Meaning:** Further extensions required to fulfill request
- **Use Case:** Client needs to send extension
- **Rarely Used**

## **511 Network Authentication Required**
- **Meaning:** Client needs to authenticate to gain network access
- **Use Case:** Captive portals (WiFi login pages)
- **Example:** Hotel/Airport WiFi login
- **Common:** Public WiFi networks

# 🔍 Best Practices

## Choosing the Right Status Code

1. **Success Operations:**
   - GET → 200 OK
   - POST (create) → 201 Created
   - PUT (update) → 200 OK or 204 No Content
   - DELETE → 204 No Content
   - PATCH → 200 OK or 204 No Content

2. **Authentication:**
   - Missing credentials → 401 Unauthorized
   - Invalid credentials → 401 Unauthorized
   - Valid user but no permission → 403 Forbidden

3. **Validation:**
   - Malformed request → 400 Bad Request
   - Valid format but business rule violation → 422 Unprocessable Entity

4. **Not Found:**
   - Resource never existed or unknown → 404 Not Found
   - Resource existed but deleted → 410 Gone (optional)

5. **Server Errors:**
   - Unknown error → 500 Internal Server Error
   - Maintenance → 503 Service Unavailable
   - Backend timeout → 504 Gateway Timeout

## Security Considerations

```csharp
// ❌ DON'T expose internal errors
catch (Exception ex)
{
    return StatusCode(500, ex.ToString()); // Leaks stack trace!
}

// ✅ DO return generic errors
catch (Exception ex)
{
    _logger.LogError(ex, "Error processing request");
    return StatusCode(500, new { error = "An error occurred" });
}

// ❌ DON'T reveal if user exists
if (userNotFound)
    return NotFound("User not found");
else
    return Unauthorized("Invalid password");

// ✅ DO return same error for both cases
if (userNotFound || invalidPassword)
```

- 401 vs 403 distinction is important
- Don't expose sensitive error details
- Implement proper logging alongside status codes
- Rate limiting with 429 is crucial
